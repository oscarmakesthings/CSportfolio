# CSportfolio

How do I approach designing software?

I approach designing software by focusing on the main goal of the project. I'll then break that goal down into smaller parts. For my 3D scene specifically, I started by thinking about what and shapes I wanted to use to create a scene. I broke down the parts of the required features that I needed to work on. Then worked on each object one by one. This helped me completed the project without becoming overwhelmed.

What new design skills has your work on the project helped you to craft?

This project helped me to think visually. I've done some design work in the past (graphic and web). But, I learned how to plan a design in a 3D environment using different variables (like shapes, coordinates, scale, and texture). Instead of only thinking about whether the code worked and was clean, I thought about how the final scene looked, and how it would be experienced.

What design process did you follow for your project work?

The process was a lot of experimentation. Changing a few small variables or placement of code could change the whole scene drastically. This caused me to do a lot of trial and erorr to complete the project. First I created an object, then I would experiement with it until I was happy with how it looekd. Then I adjusted their size, position, and rotation depending on how they worked within the scene. After that, adding lighting and texture was a big part of shaping the overall feel of the scene. I tested each change as I went so I could see how it affected the scene.

How could tactics from your design approach be applied in future work?

I feel that the same process applies to alot of software development. It's easy to get lost in the code and not see how all of the different parts make up the whole. By starting with a clear and main goal and breaking it down into smaller parts, I can take a complex project and piece it apart into something manageable. Iterating and experimenting with features or bits of code is also a useful tactic. In future work, I can use this approach for web apps, mobile projects, or other visual projects where UX is important.

What new development strategies did you use while working on your 3D scene?

I used separate functions for creating objects, loading textures, handling input, and controlling the camera. I also learned to make small changes and test them right away instead of changing too many things at once. This helped me debug problems more easily.

How did iteration factor into your development?

Iteration was a big part of my development process. There were many times frustration mounted up because things didn't work the way I expected and had to look up why. Pushing through that and just continuing to try was a big unlock for me, because I've had a lot of hard times while learning to program where the frustration was a lot. By testing the program over and over again and looking at the results, I was able to make adjustments consistently. By improving the project step by step, I was able to create a more complete and higher quality scene.

How has your approach to developing code evolved throughout the milestones, which led you to the project’s completion?

While going through the milestones, my approach became more organized and directed. At the beginning, I focused mostly on trying to build the entire scene at once. I was still learning what all of the code was doing and a bit overwhelmed with all the new bits of information to learn. As I progressed, I became more comfortable with the structure of the code and started thinking more about piecing out the program mentally. Then building it accordingly. I learned that it is easier to complete a complex project when I build and test one feature at a time. By the final project, I had a better understanding of how the different pieces of the code worked together.
